JavaScript တွေ အကြောင်းပြောရတာ Articles တစ်ခုဆို တစ်ခုနဲ့ ထိုက်တန်တဲ့ အားရမှုဖြစ်စေတာ အမှန်ပဲ ... ကျွန်တော်ကလည်း သိပ်ချစ်တတ်လွန်းတာလဲ ပါမှာပေါ့...။

JS တွေ ကို တှငျတှငျကယြျကယြျနဲ့ အသုံးပွုလာကွတာ နှစျပေါငျး ၂၀ နဲ့ အထကျရှိလာခဲ့ပါပွီ...။ တကယျတော့ web developer တှကေ HTML နဲ့ CSS ကို အထောကျအပံ့ ဖွစျစရေုံလောကျသာ JS ကို တှဲသုံးခဲ့ကွတာပါ... အဲ့လို programming language မြိုးလေးပေါ့...။

ပထမ စပြောရမှာက JS ရဲ့ language အမျိုးအစားကို စပြောရပါလိမ့်မယ်... JS ဟာ Dynamic Programming Language အမျိုးအစားဖြစ်တယ်... အဓိက ကတော့ interactive web page တွေကို ဖန်တီးတဲ့နေရာမှာ အသုံးပြုရတယ်...။

ဘာကြောင့် JS ကို သုံးရလဲ ဆိုတော့ web စာမျက်နှာ တစ်ခုမှာ ဘယ်အချိန် ဘာဖြစ်ရမယ် ဘာကတော့ မဖြစ်ရဘူး စတဲ့ ဆုံးဖြတ်ချက်မျိုးကို JS က ချမှတ်နိုင်ပြီး HTML နဲ့ CSS က အဲ့ဒီလို မဆောင်ရွက်နိုင်လို့ပါပဲ...(ဥပမာ login form တစ်ခုမှာ user name or email ထည့်ရမယ့် text box ကို ကျော်မိတာ နဲ့ user ကို information ပေးနိုင်တာမျိုး..)။

သို့သော် ခုလိုအောင်မြင်နေတဲ့ခေတ်ကြီးထဲမှာ web design project အတွက် JS ကို pure ထိုင်မရေးကြတော့တာလဲ သင်သတိထားမိပါလိမ့်မယ်... သူတို့တွေ JS Framework တွေ သုံးလာကြပြီဗျ...။ မသုံးရင်ရော လို့ သင် မေးရင် သင်အများကြီးလွတ်သွားမှာပေါ့လို့ပဲ ဖြေပါရစေ...။

ဒါဆို JS Framework တွေကိုရော ဘာလို့သုံးကြတာလဲ...?
ဥပမာလေး တစ်ခုပေးပြီးရှင်းပြရမယ်ဆို ဓါတ်လှေခါးပါတဲ့ တိုက်တစ်ခုရဲ့ အလွှာ ၂၀ ကို သင် ဓါတ်လှေကားနဲ့ တက်မလား ဒါမှမဟုတ် ရိုးရိုးလှေကားနဲ့ တက်မလား... သင့်အတွက် လွယ်ကူသက်သာတာကို သင်ပိုသဘောကျမှာပါ... ဒါကြောင့် web designers အတော်များများဟာ JS Framework တွေနဲ့ လုပ်ရတာကိုပိုချစ်ကြပါတယ်...။ တိုတိုတုတ်တုတ် ပြောရမယ်ဆို သင်က JS Framework တစ်ခုခုကို သုံးမယ်ဆိုတာနဲ့ ငွေကုန်ကြေးကျ သက်သာမယ်, အချိန်ကုန်သက်သာမယ်, အားစိုက်ရတာ သက်သာမယ်...။

အဲ့တော့ popular ဖြစ်ပြိး free လဲရတဲ့ JS Framework တွေနဲ့ ၄င်းတို့ရဲ့ အားသာချက် အချို့ကိုဆွေးနွေးပေးသွားပါမယ်...။

No.4 - Node Js
Node က chrome browser ရဲ့ JS Engine ကို base လုပ်ပြီးတည်ဆောက်ထားတာပါ...။ Network app တွေနဲ့ server-side app တွေ အတွက်ဆိုရင်တော့ Node ကိုရွေးသင့်တယ်... back-end service နဲ့ API တွေ ဖန်တီးတဲ့နေရာမှာလဲ တကယ်ကောင်းတဲ့ JS Framework တစ်ခုပါ...ဒီထက် ပိုကောင်းတာ တစ်ခုက Node ဟာ data streaming တွေ အတွက်တော့ ဘုရင် တစ်ပါးပါပဲ…။

Node နဲ့ ရေးတဲ့ web app တွေဟာ OS မရွေးဘူး … window လား mac လား linux လား ဘာလာလာ ဒေါင်းတယ်…။ ဒါကြောင့် realtime web app တွေ ရေးမယ်ဆို node ကတော့ ဆရာကြီးပဲ…။ Node ကို သုံးတဲ့ webapp တွေကတော့ Netflix တို့ Godaddy တို့ပေါ့ဗျာ…။

No.2 – Angular Js
ကျွန်တော့်ရဲ့ Career နဲ့ ကိုက်လို့လား မသိဘူး… သိပ်သဘောကျတဲ့ JS Framework တွေထဲက တစ်ခုပေါ့…။ ထွက်ခဲ့သမျှ Framework တွေထဲမှာ powerful လဲ ဖြစ်တယ်… efficient လဲ ဖြစ်တယ် ဆိုတဲ့ ဂုဏ်လေးတွေ ပိုင်ဆိုင်ထားတဲ့ Framework တစ်ခုပါ…။

Angular ကို 2009 ခုနှစ်မှာ Google က စတင်မိတ်ဆက်ခဲ့ပါတယ်…။ ထွက်တဲ့နှစ်က စလို့ ခုထက်ထိ Angular ကို Google က နှစ်တိုင်းနှစ်တိုင်း Take Care လုပ်တာ က သတိထားမိလာစေတာပဲ…။

Angular ကို Dynamic Web App တွေ ရေးမယ်ဆိုရင် အသုံးသင့်ဆုံး Framework တစ်ခုပဲ… Angular သုံးကြတဲ့ ရည်ရွယ်ချက်တွေထဲမှာ Highlight ဖြစ်တဲ့ အရာ တစ်ခုက Multiple Form တွေအကြားမှာ Robustness ကောင်းကောင်းနဲ့ အလုပ်လုပ်နိုင်ခြင်းပဲ ဖြစ်တယ်…။ ကမ္ဘာပေါ်မှာ လက်ရှိ Angular သုံးတဲ့ websites ပေါင်း 350000 ကျော် ရှိတယ်… ဒါကြောင့် Angular က popular ဖြစ်တဲ့ Framework ထဲ ပါတာ အ့ံသြစရာတော့ မဟုတ်ခဲ့ပါဘူး…။( ကျန်ခဲ့မှာ စိုးလို့… Angular က ၆ လ တစ်ခါ New Feature တွေ ထွက်တယ်လို့ သိရတယ်…)

No.1 – React Js
ဒီနာမည်ကို ဒီဘက်ခေတ် developer အလုပ်ခေါ်စာတွေမှာ မကြာခဏ ဒါမှမဟုတ် ခဏခဏ တွေ့နေရပါလိမ့်မယ်… ဒါက သူ့ကို Popular ဖြစ်တယ်ဆိုတာ သိပ် အမွှန်းတင်ဖို့ လိုမယ်မထင်ဘူး…။

2013 ခုနှစ်မှာ Jordan Walke ဆိုတဲံ Facebook က Software Engineer တစ်ယောက်က စတင် မိတ်ဆက်ခဲ့တာပဲ ဖြစ်တယ်…။ ယေဘုယျအားဖြင့်တော့ React ကို single page web app တွေအတွက် အသုံးများတယ်… သို့သော် … React ဟာ UI components တွေကို Reusable လုပ်ပေးနိုင်သေးသလို data တွေကိုလည်း automatically update and render လုပ်နိုင်သေးတယ်… (ဒါကမှ သူ့ ရဲ့ အားသာချက်…)။

ဒါ့အပြင် React ကို Mobile app (IOS, Android) တွေအတွက် မကြာခဏ ဆိုသလို အသုံးများတာ တွေ့ရတယ်… React Js ကို အသုံးပြုထားတဲ့ Websitesပေါင်း 475000 ကျော် ရှိပြီး ဒီထက် ပိုတိုးလာဖို့ ပဲ ရှိတယ်လို့ မြင်တယ်…။
“React ကတော့ React ပါပဲ” …။
